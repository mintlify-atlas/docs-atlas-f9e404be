---
title: 'List pushes'
description: 'Retrieve paginated lists of your active or expired pushes.'
---

Two endpoints let you retrieve pushes associated with your account: one for pushes that are still active and one for pushes that have expired.

Both endpoints require authentication. Anonymous pushes are not linked to an account and will not appear in these results.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/p/active.json` | List active (non-expired) pushes |
| `GET` | `/p/expired.json` | List expired pushes |

## Query parameters

Both endpoints accept the same pagination parameter.

<ParamField query="page" type="integer" default="1">
  Page number to retrieve. Minimum `1`, maximum `200`. Returns up to 50 pushes per page, ordered by creation date descending.
</ParamField>

## Response

Both endpoints return an array of push objects.

<ResponseField name="url_token" type="string">
  The unique token for the push.
</ResponseField>

<ResponseField name="html_url" type="string">
  Fully qualified URL for viewing the push in a browser.
</ResponseField>

<ResponseField name="json_url" type="string">
  Fully qualified URL for retrieving the push via the API.
</ResponseField>

<ResponseField name="expired" type="boolean">
  Whether the push has expired.
</ResponseField>

<ResponseField name="deleted" type="boolean">
  Whether the push was manually expired.
</ResponseField>

<ResponseField name="expire_after_days" type="integer">
  The configured days-until-expiration setting.
</ResponseField>

<ResponseField name="expire_after_views" type="integer">
  The configured views-until-expiration setting.
</ResponseField>

<ResponseField name="days_remaining" type="integer">
  Computed days remaining before expiration.
</ResponseField>

<ResponseField name="views_remaining" type="integer">
  Computed views remaining before expiration.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 creation timestamp.
</ResponseField>

<ResponseField name="expires_at" type="string">
  ISO 8601 timestamp for date-based expiration.
</ResponseField>

<ResponseField name="name" type="string">
  Dashboard label, if set.
</ResponseField>

<ResponseField name="note" type="string">
  Private creator note, if set.
</ResponseField>

## Examples

### Active pushes

<CodeGroup>

```bash cURL
curl -X GET \
  -H "X-User-Email: user@example.com" \
  -H "X-User-Token: MyAPIToken" \
  https://pwpush.com/p/active.json
```

```bash cURL (page 2)
curl -X GET \
  -H "X-User-Email: user@example.com" \
  -H "X-User-Token: MyAPIToken" \
  "https://pwpush.com/p/active.json?page=2"
```

```python Python
import requests

response = requests.get(
    "https://pwpush.com/p/active.json",
    headers={
        "X-User-Email": "user@example.com",
        "X-User-Token": "MyAPIToken",
    },
)
print(response.json())
```

```ruby Ruby
require "net/http"
require "json"
require "uri"

uri = URI("https://pwpush.com/p/active.json")
http = Net::HTTP.new(uri.host, uri.port)
http.use_ssl = true

request = Net::HTTP::Get.new(uri)
request["X-User-Email"] = "user@example.com"
request["X-User-Token"] = "MyAPIToken"

response = http.request(request)
puts JSON.parse(response.body)
```

## Error responses

```json 401 Unauthorized
{}
```

```json 400 Bad Request
{
  "error": "Invalid page parameter"
}
```

The `400` response is returned when `page` is not a valid integer or exceeds the maximum of 200.
