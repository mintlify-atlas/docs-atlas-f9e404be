The official Docker image for Password Pusher is published to Docker Hub and supports Linux/amd64 and Linux/arm64. It bundles a web server and an optional background worker managed by Foreman, so a single container handles both in the default configuration.

<Note>
  The image exposes ports `80`, `443`, and `5100`. Port `5100` is the internal Puma/Thrust port and is only needed if you are not using TLS. If you set `TLS_DOMAIN`, port `5100` can be removed from the port mapping.
</Note>

## Image tags

| Tag      | Description                                       |
| -------- | ------------------------------------------------- |
| `latest` | Most recent build — generally reliable            |
| `stable` | Well-tested releases — recommended for production |

```
docker.io/pglombardo/pwpush:latest
docker.io/pglombardo/pwpush:stable
```

## Quick start with Docker Compose

The repository ships a `docker-compose.yml` that covers the most common deployment scenario: a single container with a named volume for persistent SQLite storage and file uploads.

<Steps>
  <Step title="Copy the compose file">
    Download or copy the `docker-compose.yml` from the repository root to your server.

    ```bash
    curl -O https://raw.githubusercontent.com/pglombardo/PasswordPusher/master/docker-compose.yml
    ```
  </Step>

  <Step title="Configure environment variables">
    Open `docker-compose.yml` and set your configuration under the `x-env` block. At minimum, consider setting `PWPUSH_MASTER_KEY` for production deployments and `TLS_DOMAIN` if you want automatic HTTPS.

    ```yaml
    # PWPUSH_MASTER_KEY: ""   # Generate at https://us.pwpush.com/generate_key
    # TLS_DOMAIN: ""          # e.g. pwpush.example.com — provisions Let's Encrypt certs automatically
    ```

    Alternatively, uncomment `env_file: - .env` and place your variables in a `.env` file.
  </Step>

  <Step title="Start the container">
    ```bash
    docker compose up -d
    ```
  </Step>

  <Step title="Access the application">
    - **HTTP**: `http://localhost` (port 80)
    - **HTTPS**: `https://localhost` (port 443, requires `TLS_DOMAIN`)
    - **Direct Puma**: `http://localhost:5100`

    Confirm the container is healthy:

    ```bash
    docker compose ps
    curl -f http://localhost:5100/up
    ```
  </Step>
</Steps>

## Key environment variables

The table below covers the variables most commonly needed when first setting up a deployment. The full list of all `PWP__` options is documented in the [self-hosted configuration reference](https://docs.pwpush.com/docs/self-hosted-configuration/).

| Variable             | Default | Description                                                                                                  |
| -------------------- | ------- | ------------------------------------------------------------------------------------------------------------ |
| `PWPUSH_MASTER_KEY`  | —       | Encryption key for stored payloads. Omitting this uses a built-in default. Generate at `/generate_key`.      |
| `TLS_DOMAIN`         | —       | Domain name for automatic Let's Encrypt TLS provisioning (e.g. `pwpush.example.com`).                       |
| `PWP__ALLOW_ANONYMOUS` | `true` | Allow push creation without signing in.                                                                     |
| `PWP__ENABLE_URL_PUSHES` | `true` | Enable URL push type.                                                                                   |
| `PWP__ENABLE_FILE_PUSHES` | `true` | Enable file push type.                                                                                 |
| `PWP__ENABLE_QR_PUSHES` | `true` | Enable QR code push type.                                                                               |
| `PWP__FILES__STORAGE` | `local` | File storage backend: `local`, `s3`, `gcs`, or `azure`.                                                  |
| `PWP__NO_WORKER`     | —       | Set to `true` to run the web process only, without the background worker.                                    |

## Persistent storage

The named volume `pwpush-storage` maps to `/opt/PasswordPusher/storage` inside the container. This directory holds:

- The SQLite database (`storage/db/`)
- Uploaded files when `PWP__FILES__STORAGE` is set to `local`

To use a host path instead of a named volume, replace the volume entry in `docker-compose.yml`:

```yaml
volumes:
  - /path/on/host:/opt/PasswordPusher/storage
```

<Warning>
  Removing the volume or changing the mount path without migrating data will result in data loss. The SQLite database lives inside this directory.
</Warning>

## Health check

The container's built-in health check calls the `/up` endpoint every 30 seconds:

```yaml
healthcheck:
  test: ["CMD", "curl", "-f", "http://localhost:5100/up"]
  interval: 30s
  timeout: 10s
  retries: 3
  start_period: 40s
```

You can verify health at any time:

```bash
curl -f http://localhost:5100/up
```

## Using TLS / HTTPS

Set `TLS_DOMAIN` to a fully-qualified domain name that already resolves to your server's IP. The container will provision and renew a Let's Encrypt certificate automatically.

```yaml
TLS_DOMAIN: "pwpush.example.com"
```

Once `TLS_DOMAIN` is set, port `5100` is no longer needed externally and can be removed from the port mapping:

```yaml
ports:
  - "80:80"
  - "443:443"
  # - "5100:5100"  # not needed when TLS_DOMAIN is set
```

## Running without a compose file

You can also run the container directly with `docker run`:

```bash
docker run -d \
  --name pwpush \
  --restart unless-stopped \
  -p 80:80 -p 443:443 -p 5100:5100 \
  -v pwpush-storage:/opt/PasswordPusher/storage \
  -e PWPUSH_MASTER_KEY="your-key-here" \
  docker.io/pglombardo/pwpush:stable
```
