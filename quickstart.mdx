Choose your path: use the hosted service at pwpush.com with no setup, or run your own instance with Docker Compose.

<Tabs>
  <Tab title="Hosted (pwpush.com)">
    The hosted service requires no installation. You can create and share a push in under a minute.

    <Steps>
      <Step title="Open pwpush.com">
        Navigate to [pwpush.com](https://pwpush.com) in your browser. You do not need an account to create a push.
      </Step>
      <Step title="Enter your secret">
        Type or paste the text you want to share into the input field. Optionally:
        - Set how many views the link allows before it expires (default: 5)
        - Set how many days before it expires (default: 7)
        - Add a passphrase that the recipient must enter to unlock the secret
        - Enable a "retrieval step" so the recipient must click through a confirmation page first
      </Step>
      <Step title="Click Push it!">
        Click the **Push it!** button. Password Pusher encrypts your secret and returns a unique one-time URL.
      </Step>
      <Step title="Share the link">
        Copy the generated link and send it to the recipient through email, chat, or any channel. The link is the only way to retrieve the secret.
      </Step>
      <Step title="Link auto-expires">
        After the configured number of views or days — whichever comes first — the secret is permanently deleted. Anyone who arrives after that sees only an "expired" page.
      </Step>
    </Steps>

    <Tip>
      Sign up for a free account to unlock audit logs showing every view event: IP address, browser, timestamp, and referrer.
    </Tip>
  </Tab>

  <Tab title="Self-hosted (Docker Compose)">
    Run Password Pusher on your own server with a single Docker Compose file. The image includes a built-in SQLite database, file storage, and optional automatic TLS via Let's Encrypt.

    <Steps>
      <Step title="Point DNS to your server">
        Create an A record (or AAAA for IPv6) pointing your domain to the public IP of the server where you will run the container. For example:

        ```
        pwpush.example.com  A  203.0.113.10
        ```

        This step is required only if you want automatic HTTPS. If you are evaluating locally, skip this step and access the app on port 5100.
      </Step>
      <Step title="Download docker-compose.yml">
        Clone the repository or download the Compose file directly:

        ```bash
        curl -O https://raw.githubusercontent.com/pglombardo/PasswordPusher/refs/heads/master/docker-compose.yml
        ```

        The full file contents:

        ```yaml docker-compose.yml
        # Password Pusher - Docker Compose

        x-env: &x-env
          environment:
            # PWPUSH_MASTER_KEY: ""   # Generate at https://us.pwpush.com/generate_key
            # TLS_DOMAIN: ""          # e.g. pwpush.example.com

            PWP__ENABLE_URL_PUSHES: "true"
            PWP__ENABLE_FILE_PUSHES: "true"
            PWP__ENABLE_QR_PUSHES: "true"
            PWP__ALLOW_ANONYMOUS: "true"
            PWP__FILES__STORAGE: "local"

            # PWP__HOST_DOMAIN: ""
            # PWP__HOST_PROTOCOL: "https"
            # PWP__DISABLE_SIGNUPS: "false"
            # ...

        services:
          pwpush:
            image: docker.io/pglombardo/pwpush:latest
            restart: unless-stopped
            ports:
              - "80:80"
              - "443:443"
              - "5100:5100"
            volumes:
              - pwpush-storage:/opt/PasswordPusher/storage
            healthcheck:
              test: ["CMD", "curl", "-f", "http://localhost:5100/up"]
              interval: 30s
              timeout: 10s
              retries: 3
              start_period: 40s
            <<: *x-env

        volumes:
          pwpush-storage:
            driver: local
        ```
      </Step>
      <Step title="Set TLS_DOMAIN and (recommended) PWPUSH_MASTER_KEY">
        Open `docker-compose.yml` in an editor and uncomment the two key lines under `x-env`:

        ```yaml
        PWPUSH_MASTER_KEY: "your-generated-key-here"
        TLS_DOMAIN: "pwpush.example.com"
        ```

        - **`TLS_DOMAIN`** — The domain you pointed DNS to in step 1. The container automatically provisions and renews a Let's Encrypt certificate.
        - **`PWPUSH_MASTER_KEY`** — The encryption key used to protect secrets at rest. Generate one at [us.pwpush.com/generate_key](https://us.pwpush.com/generate_key). If omitted, a built-in default key is used (fine for evaluation, not recommended for production).

        <Warning>
          If you change `PWPUSH_MASTER_KEY` after pushes have been created, existing encrypted data will become unreadable. Set it once before going live and do not rotate it without a migration plan.
        </Warning>
      </Step>
      <Step title="Start the container">
        From the directory containing `docker-compose.yml`, run:

        ```bash
        docker compose up -d
        ```

        The container starts, runs database migrations, and begins serving traffic. The first boot may take 30–40 seconds for initialization. Check readiness with:

        ```bash
        docker compose logs -f pwpush
        ```
      </Step>
      <Step title="Open your instance">
        - **With TLS_DOMAIN set:** Open `https://pwpush.example.com`
        - **Without TLS_DOMAIN:** Open `http://your-server-ip:5100`

        You should see the Password Pusher home page. Create your first push to verify everything is working.

        <Tip>
          The `pwpush-storage` Docker volume persists the SQLite database and uploaded files across container restarts and upgrades. To use a host directory instead, replace the volume with `- /path/on/host:/opt/PasswordPusher/storage`.
        </Tip>
      </Step>
    </Steps>
  </Tab>
</Tabs>
